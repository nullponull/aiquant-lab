## マガジン説明文

note の連載マガジン作成時の説明文：

```
「AI で市場予測は無理」と言われ続けている領域に、
実装でどこまで迫れるかを 6 つの実験で確かめる連載。

論文では理論が示されている。
ただ現代の AI で実装して、過去データと公開リソースで
体系的に試した記録は、日本語ではほぼ存在しない。

そこを埋めたい。

毎週水曜 12:00 更新。全コード GitHub 公開、
失敗も含めてすべて記録。

【6 つの壁】
- 再帰性: 戦略は知られた瞬間死ぬ
- 非定常性: 過去 ≠ 未来の統計分布
- グッドハート: 最適化が意味を失う
- 複雑系: カオスで予測不可
- ファットテール: 過去にない事象
- 自己言及性: 価格が自分を含む

各壁に対して 1 つずつ実装で挑戦します。

※本マガジンは教育・研究目的の情報提供であり、
投資助言ではありません。
```

## 既存ファネル記事への自動挿入手順

`daily-note-post` プロジェクトの記事生成テンプレートに、上記「バージョンA」を末尾に自動挿入する設定を追加：

### 実装メモ

1. `daily-note-post/templates/article_template.md` に末尾セクションを追加
2. 環境変数 `LATEST_QUANT_ARTICLE_URL` を取得して動的にリンクを差し込む
3. 連載が進むたびに最新記事を更新

これにより、毎日のファネル記事がすべて連載へのトラフィック源になります。

## URL プレースホルダーの置換ルール

実投稿時に置換する URL リスト：

| プレースホルダー | 実 URL |
|--------------|------|
| `[マニフェストURL]` | note 記事公開後の URL |
| `[第1回URL]` | note 記事公開後の URL |
| `[マガジンURL]` | note マガジン作成後の URL |
| `[GitHubURL]` | リポジトリ公開後の URL |

実投稿前に grep で置換確認すること:
```bash
grep -r "URL]" /home/sol/aiquant-lab/
```
