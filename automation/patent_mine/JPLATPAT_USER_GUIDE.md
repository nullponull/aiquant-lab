# J-PlatPat CSV エクスポート手順 (人間操作 + 自動処理ハイブリッド)

> J-PlatPat は Angular SPA で大規模自動スクレイピングが困難。
> しかし**人間が CSV エクスポート → 自動処理パイプライン**は完全に動作する。
> これは合法的（規約に整合）かつ確実な方法。

---

## なぜこの方法を採用するか

| 方法 | 法的 | 確実性 | 速度 |
|------|------|------|------|
| Google Patents 自動 | グレー (規約) | × (Sorry 検出) | - |
| J-PlatPat 自動 | グレー (規約) | × (Angular 困難) | - |
| **CSV エクスポート + 自動処理** | **◎** | **◎** | **○** |
| EPO API | ◎ | ○ | ○ (要 API キー) |

J-PlatPat 規約: 個人利用・研究利用は OK、機械的大量取得は不可。
**CSV エクスポート機能は規約上正規の入手方法**。

---

## ステップバイステップ

### Step 1: J-PlatPat で検索

1. ブラウザで https://www.j-platpat.inpit.go.jp/s0100 を開く
2. 「**特許・実用新案**」を選択（ラジオボタン）
3. 検索キーワード入力（推奨例）:

| カテゴリ | キーワード例 |
|---------|----------|
| キッチンツール | 「キッチン 便利」「料理 補助」「調理 簡易」 |
| ペット用品 | 「ペット 給水」「犬 玩具」「猫 トイレ」 |
| 介護用品 | 「介護 補助」「立ち上がり」「歩行 支援」 |
| 文房具 | 「文房具 構造」「ペン 持ち手」 |
| DIY 工具 | 「DIY 工具」「片手 クランプ」 |
| 収納 | 「収納 折り畳み」「省スペース」 |
| 育児 | 「育児 便利」「子育て 補助」 |

**重要**: キーワードは具体的に。「便利」だけだと結果多すぎ → 検索失敗

4. 検索ボタンをクリック

### Step 2: 期限切れに絞り込み (重要)

1. 検索結果ページの「**詳細条件**」「**絞り込み**」を展開
2. 「公報発行日」または「公開日」で **2000-01-01 〜 2005-12-31** を指定
   - 20 年以上前の公開 = 特許権満了済み（高確率）
3. 必要なら「経過情報」で「**消滅**」「**期間満了**」を選択

### Step 3: CSV エクスポート

1. 検索結果が表示されたら、画面上部の「**CSV出力**」ボタンをクリック
2. 出力項目を選択:
   - ✅ 文献番号
   - ✅ 出願番号
   - ✅ 出願日
   - ✅ 公開日 (公報発行日)
   - ✅ 発明の名称
   - ✅ 出願人/権利者
   - ✅ 発明者
   - ✅ IPC
   - ✅ 経過状態 (権利状況)
   - ✅ 要約 (取得できる範囲で)

3. ダウンロードされた CSV ファイル (例: `search_results.csv`) を保存

### Step 4: 自動処理パイプラインに投入

```bash
cd /home/sol/aiquant-lab

# CSV をプロジェクトに移動
cp ~/Downloads/search_results.csv automation/patent_mine/data/

# Claude スコアリング実行
uv run python automation/patent_mine/run_from_csv.py \
    automation/patent_mine/data/search_results.csv \
    --max 50 \
    --output-suffix kitchen
```

### Step 5: 結果確認

```bash
# 生成されたショートリスト
cat automation/patent_mine/results/shortlist_$(date +%Y-%m-%d)_kitchen.md

# JSON データ (詳細)
cat automation/patent_mine/results/scored_$(date +%Y-%m-%d)_kitchen.json | jq '.[] | select(.category=="viable")' | head -50
```

## 効率的な検索戦略

### 月 1000 件処理を目指す場合

1 検索で 100-200 件取得 → 月 5-10 検索（カテゴリ別） = 1000 件達成

| カテゴリ | 1 回の取得目安 | コスト (Claude) | viable 想定 |
|--------|------------|------------|-----------|
| キッチン | 100 件 | $0.10 | 5-15 件 |
| ペット | 100 件 | $0.10 | 8-20 件 |
| 介護 | 100 件 | $0.10 | 5-15 件 |
| 文房具 | 100 件 | $0.10 | 3-10 件 |
| DIY | 100 件 | $0.10 | 5-15 件 |
| 収納 | 100 件 | $0.10 | 5-12 件 |
| 育児 | 100 件 | $0.10 | 5-12 件 |
| 計 | **700 件** | **$0.70** | **36-99 件 viable** |

→ 1 ヶ月でショートリスト 30-100 件、その中で「魅力的」が 5-10 件

### CSV エクスポートの上限

J-PlatPat は 1 検索につき**最大 1000 件まで**エクスポート可能。それ以上は絞り込み必要。

## トラブルシューティング

### CSV 出力ボタンがない

- ログインが必要な場合がある（個人アカウント、無料）
- 検索結果が 0 件だと出ない

### CSV を開いたら文字化け

- `cp932` (Shift-JIS) で出力されている可能性
- `jplatpat_csv_loader.py` は自動判定するので問題ない
- Excel で開きたい場合は UTF-8 BOM 付きで再保存

### 「期限切れ」と出るのに権利存続中の特許が混ざる

- ステータスベース判定だけだと不完全
- 弁理士に再確認させる（Phase 1 の本来の役割）

### 大量検索で BAN される

- J-PlatPat は明確な人間利用想定
- 1 日 5-10 検索なら問題ない
- 10 検索/日 を超えたら翌日にリセット

## 弁理士フェーズへの引き継ぎ

ショートリストの上位 5-10 件について:

1. `shortlist_*.md` を弁理士に送る
2. 各候補について `PHASE1_LEGAL_CLEARANCE.md` のチェックリストで確認依頼
3. CLEAR 判定の 3-5 件で Phase 2 (試作) 進む

弁理士費用: 1 件あたり ¥10,000-30,000、5 件で ¥75,000-150,000

## 自動化の限界と現実解

| 工程 | 自動化可能 | 工数 |
|------|--------|------|
| キーワード検索 | ✗ (手動) | 5 分/カテゴリ |
| CSV エクスポート | ✗ (手動) | 1 分/カテゴリ |
| Claude スコアリング | ✅ | 10 分/100 件 (auto) |
| ショートリスト生成 | ✅ | 自動 |
| 弁理士確認 | ✗ (専門家) | 1 件 1-3 日 |

**一日の作業**: 5 カテゴリ検索 + CSV エクスポート = 30 分の人間作業
+ 50 分の自動処理 = **1 日でショートリスト 30-100 件**生成可能。

これは note_ai_mousigo の手法を企業として運用する実用解。
